# Tan Chin MIM PMC System 五路線驗證藍圖

建立日期：2026-05-11  
目的：用五條故事化驗證路線，一條一條測試系統在實際現場情境、錯誤資料、資料異動與排程壓力下的穩定度與應對能力。  
適用範圍：桌面版 Electron 主系統，包含主要資料、設備設定、浸泡式脫脂、溶劑更換、真空式脫脂、真空式燒結、查詢報表、儀表板與系統管理。

## 1. 驗證總原則

### 1.1 測試節奏

五條路線建議依序執行：

1. 路線 A：主資料建置與空白預設驗證
2. 路線 B：設備、浸泡式脫脂與溶劑更換驗證
3. 路線 C：真空式脫脂排程與推薦驗證
4. 路線 D：真空式燒結排程與承載限制驗證
5. 路線 E：資料生命週期、異常情境與穩定性驗證

前一條路線產生的資料，會成為後一條路線的基礎。若中途資料庫被清空，請從路線 A 重新建立必要主檔。

### 1.2 共通通過標準

每一條路線都必須符合以下標準：

- 操作過程中頁面不得白屏、卡死或無限載入。
- 必填欄位未填時，系統必須阻擋儲存，且資料不得寫入資料庫。
- 儲存成功後，列表、搜尋、報表或儀表板應能反映新增資料。
- 重新整理資料或重開程式後，已儲存資料仍存在。
- 下拉選單未選擇時，應維持「請選擇...」類型的預設提示。
- 編輯既有資料時，欄位應正確帶入原資料，不應被空白預設覆蓋。
- 操作失敗時，系統應顯示可理解的錯誤訊息，不應只在 console 出錯。
- 日期、數字、容量、高度、堆疊數等欄位不可出現 NaN、undefined、null 文字外露。

### 1.3 證據留存建議

每跑完一條路線，建議留下以下紀錄：

- 測試日期與測試人員。
- 使用版本或打包檔版本。
- 測試資料代碼，例如 `QA-PM-A001`、`QA-JIG-A001`。
- 每個重大節點的截圖。
- 失敗步驟的實際輸入、畫面訊息與重現步驟。
- 若有資料庫備份，標記備份時間點。

### 1.4 測試資料命名規則

為避免與正式資料混淆，所有驗證資料建議使用 `QA-` 前綴。

| 類型 | 命名範例 | 用途 |
| --- | --- | --- |
| 產品主檔 | `QA-PM-A001` | 主資料與作業標準測試 |
| 作業標準 | `QA-SPEC-A001` | 真空脫脂與燒結排程測試 |
| 治具 | `QA-JIG-TRAY-A001` | 承載治具測試 |
| 設備 | `QA-MCH-DEG-A001` | 脫脂設備測試 |
| 批次 | `QA-BATCH-VAC-A001` | 真空排程批次測試 |

## 2. 路線 A：主資料建置與空白預設驗證

### 2.1 故事情境

工程人員第一次進入「主要資料」，準備建立新產品、作業標準、治具資料，並把治具與作業標準建立對應關係。此路線確認系統在沒有預設值誤導的狀態下，能正確要求使用者選擇與填寫資料。

### 2.2 驗證目標

- 產品資料、作業標準、治具與對應標準頁面進入時應呈現空白或「請選擇...」狀態。
- 新增資料流程可完整建立：產品主檔 -> 作業標準 -> 治具 -> 治具對應規則。
- 必填欄位、下拉欄位、數字欄位能正確阻擋不完整資料。
- 搜尋、編輯、重置表單不會破壞資料。

### 2.3 前置條件

- 系統可正常啟動。
- 若已有正式資料，先備份資料庫或使用測試環境。
- 目前至少可以開啟「主要資料」頁面。

### 2.4 建議測試資料

| 資料類型 | 欄位 | 值 |
| --- | --- | --- |
| 產品主檔 | 產品代碼 | `QA-PM-A001` |
| 產品主檔 | 產品名稱 | `驗證用齒輪胚件 A` |
| 產品主檔 | 主檔版次 | `R01` |
| 作業標準 | 對應產品資料 | `QA-PM-A001` |
| 作業標準 | 作業標準代碼 | `QA-SPEC-A001` |
| 作業標準 | 作業標準名稱 | `標準排程驗證 A` |
| 作業標準 | 製程版次 | `A1` |
| 作業標準 | 產品高度 | `18.5` |
| 作業標準 | 單盤容量 | `120` |
| 治具 | 治具編號 | `QA-JIG-TRAY-A001` |
| 治具 | 治具名稱 | `驗證石墨托盤 A` |
| 治具 | 治具材質 | 石墨 |
| 治具 | 用途 | 承載 |
| 治具 | 高度 | `12` |
| 治具 | 最大堆疊數 | `3` |

### 2.5 操作步驟與預期結果

| 步驟 | 操作 | 預期結果 |
| --- | --- | --- |
| A-01 | 開啟「主要資料 > 產品資料」 | 表單可正常顯示，新增模式欄位不帶入上一筆資料 |
| A-02 | 不填產品代碼直接儲存 | 系統阻擋儲存，資料列表不新增空資料 |
| A-03 | 填入產品主檔測試資料並儲存 | 顯示成功訊息，產品可被搜尋或後續作業標準選取 |
| A-04 | 切換到「作業標準」 | 對應產品資料預設為「請選擇產品主檔」，其他指定欄位為空白 |
| A-05 | 不選產品主檔直接儲存 | 系統阻擋或顯示錯誤，不能產生無產品的作業標準 |
| A-06 | 選擇 `QA-PM-A001`，填入作業標準資料並儲存 | 作業標準成功建立，搜尋結果可找到 |
| A-07 | 切換到「治具與對應標準」 | 治具材質、用途、狀態、作業標準皆顯示「請選擇...」，數字欄位不自動填值 |
| A-08 | 不選治具材質或用途直接儲存 | 系統阻擋，不能建立用途不明的治具 |
| A-09 | 填入治具測試資料並儲存 | 治具成功建立，搜尋治具主檔可找到 |
| A-10 | 在製程治具對應表選擇 `QA-SPEC-A001` | 顯示該作業標準摘要與可設定治具規則 |
| A-11 | 將 `QA-JIG-TRAY-A001` 設為推薦，優先序填 `1`，儲存規則 | 顯示成功訊息，重整後規則仍存在 |
| A-12 | 編輯剛建立的產品、作業標準、治具 | 編輯模式應正確帶入原資料，不應變成空白預設 |

### 2.6 需要特別觀察

- 表單「清空並進入新增模式」是否真的清空，不保留上一筆資料。
- 從產品列表點「查看規格」時，是否仍能帶入指定產品，不受空白預設改動影響。
- 作業標準與治具對應規則儲存後，重開頁面是否仍能查到。

### 2.7 失敗判斷

- 頁面一進入就自動選到第一筆產品或第一筆作業標準。
- 未選擇下拉欄位仍可儲存成功。
- 清空表單後仍殘留上一筆資料。
- 編輯既有資料時欄位被清空。

## 3. 路線 B：設備、浸泡式脫脂與溶劑更換驗證

### 3.1 故事情境

現場人員先建立浸泡式脫脂設備，接著選擇一台或多台設備投入批次，並在設備使用一段時間後登錄溶劑更換。此路線驗證設備與浸泡式作業資料能被儀表板與報表追蹤。

### 3.2 驗證目標

- 設備可新增、編輯、依類型分表呈現。
- 浸泡式脫脂投入可選設備並建立紀錄。
- 多選設備時，系統能建立對應紀錄，不會漏寫或重複寫入。
- 溶劑更換紀錄能正確關聯設備。
- 儀表板與查詢報表能反映最新作業。

### 3.3 前置條件

- 路線 A 至少已建立一筆產品與作業標準。
- 系統可進入「設備設定」、「浸泡式脫脂作業」、「溶劑更換管理」。

### 3.4 建議測試資料

| 類型 | 欄位 | 值 |
| --- | --- | --- |
| 設備 | 設備代碼 | `QA-MCH-DEG-A001` |
| 設備 | 設備名稱 | `驗證浸泡脫脂槽 A` |
| 設備 | 設備類型 | 浸泡式脫脂設備 |
| 設備 | 狀態 | 啟用 |
| 投入批次 | 批號 | `QA-DEG-BATCH-A001` |
| 投入批次 | 作業人員 | `QA Tester` |
| 投入批次 | 產品 | `QA-SPEC-A001` |
| 溶劑更換 | 更換人員 | `QA Tester` |
| 溶劑更換 | 更換原因 | `穩定性驗證` |

### 3.5 操作步驟與預期結果

| 步驟 | 操作 | 預期結果 |
| --- | --- | --- |
| B-01 | 到「設備設定」新增浸泡式脫脂設備 | 設備出現在浸泡式脫脂設備列表 |
| B-02 | 新增第二台浸泡式脫脂設備 `QA-MCH-DEG-A002` | 多台設備可正常排序與顯示 |
| B-03 | 將其中一台改為停用 | 停用狀態應在列表與卡片上可辨識 |
| B-04 | 到「浸泡式脫脂作業」，未選設備直接送出 | 系統阻擋，提示需先選設備 |
| B-05 | 選擇啟用設備，填入批次資料並送出 | 最近投入紀錄新增資料 |
| B-06 | 多選兩台啟用設備後送出 | 兩台設備皆產生對應紀錄或依系統設計產生批次群組 |
| B-07 | 嘗試用停用設備投入 | 應阻擋或至少清楚標示不能使用 |
| B-08 | 到「溶劑更換管理」，不選設備直接送出 | 系統阻擋，不能建立無設備紀錄 |
| B-09 | 選擇設備並新增溶劑更換紀錄 | 最近更換紀錄新增，報表可查 |
| B-10 | 到「儀表板」查看浸泡式設備狀態與最近投入 | 顯示最新設備與投入資訊 |
| B-11 | 到「查詢報表」用日期與設備篩選 | 可查到 B-05、B-06、B-09 的資料 |

### 3.6 需要特別觀察

- 設備停用後，是否仍出現在作業卡片中。
- 多選設備建立資料時，成功訊息與資料筆數是否一致。
- 日期篩選是否包含當日資料。
- 溶劑更換後，設備狀態或警示是否有對應變化。

### 3.7 失敗判斷

- 未選設備仍可建立紀錄。
- 多選設備只寫入一筆或寫入重複資料。
- 儀表板不更新，必須重開才看得到。
- 報表查不到已建立的投入或更換紀錄。

## 4. 路線 C：真空式脫脂排程與推薦驗證

### 4.1 故事情境

排程人員接到多個待處理品項，要安排真空式脫脂爐。系統需要根據設備、產品高度、治具、單盤容量、批量與限制，提供可行的推薦結果。

### 4.2 驗證目標

- 真空式脫脂設備與爐體參數可被選取。
- 品項輸入可正確讀取產品作業標準。
- 治具高度、產品高度、容量與堆疊規則會影響推薦結果。
- 缺少產品、缺少治具、超過高度或容量時，系統有合理提示。
- 推薦結果可以儲存成批次，並在列表與報表中查詢。

### 4.3 前置條件

- 路線 A 已建立產品、作業標準、治具與對應規則。
- 已建立至少一台真空式脫脂設備或爐體設定。
- 若系統需要爐體 profile，需先在設備設定建立對應參數。

### 4.4 建議測試資料

| 類型 | 欄位 | 值 |
| --- | --- | --- |
| 真空設備 | 設備代碼 | `QA-MCH-VAC-A001` |
| 真空設備 | 設備名稱 | `驗證真空脫脂爐 A` |
| 批次 | 批號 | `QA-BATCH-VAC-A001` |
| 批次 | 作業人員 | `QA Tester` |
| 品項 1 | 作業標準 | `QA-SPEC-A001` |
| 品項 1 | 數量 | `240` |
| 品項 2 | 作業標準 | `QA-SPEC-A001` |
| 品項 2 | 數量 | `60` |

### 4.5 操作步驟與預期結果

| 步驟 | 操作 | 預期結果 |
| --- | --- | --- |
| C-01 | 到「設備設定」新增真空式脫脂設備 | 設備出現在真空式脫脂設備列表 |
| C-02 | 到「真空式脫脂作業」，未選設備直接建立排程 | 系統阻擋，提示需選擇真空式脫脂爐 |
| C-03 | 選擇 `QA-MCH-VAC-A001`，新增一筆正常品項 | 品項列可讀到產品高度、治具與容量資訊 |
| C-04 | 建立推薦或送出排程 | 推薦結果顯示可行方案，推薦細節不應空白 |
| C-05 | 儲存批次 `QA-BATCH-VAC-A001` | 真空式脫脂批次列表新增資料 |
| C-06 | 將品項數量調高到明顯超出容量 | 推薦結果應提示不可行或顯示限制原因 |
| C-07 | 選擇尚未設定治具規則的作業標準 | 系統應使用 fallback 或提示規則不足，不應崩潰 |
| C-08 | 使用高度極高的作業標準進行排程 | 推薦細節應指出高度限制或不可行原因 |
| C-09 | 儲存成功後到報表查詢真空式脫脂批次 | 可查到批次、品項與日期 |
| C-10 | 重開系統後回到真空式脫脂批次列表 | 已儲存批次仍存在 |

### 4.6 需要特別觀察

- 推薦結果與推薦細節是否同步更新。
- 切換設備後，舊推薦結果是否被正確清除或重新計算。
- 未完整輸入品項時是否能防止送出。
- 批次儲存後，報表與儀表板是否一致。

### 4.7 失敗判斷

- 缺少設備、批號或作業人員仍可儲存。
- 超出容量或高度限制仍被標示為可行。
- 推薦細節出現 undefined、NaN 或空白異常。
- 儲存後批次列表不顯示，或報表查不到。

## 5. 路線 D：真空式燒結排程與承載限制驗證

### 5.1 故事情境

燒結排程人員需要把已完成前段流程的品項安排進真空式燒結爐。系統需要考慮燒結爐容量、層高、治具堆疊、混裝設定與產品高度，產生可行的排盤推薦。

### 5.2 驗證目標

- 真空式燒結爐可被新增與選取。
- 燒結品項可讀取作業標準與治具資訊。
- 混裝設定、治具限制與高度限制會影響推薦。
- 推薦方案能儲存成燒結批次。
- 儀表板、報表與批次列表可追蹤燒結排程。

### 5.3 前置條件

- 路線 A 已建立至少兩個作業標準，建議一個允許混裝，一個不允許混裝。
- 已建立承載治具、墊高治具、陶瓷治具或墊腳資料。
- 已建立真空式燒結爐設備。

### 5.4 建議測試資料

| 類型 | 欄位 | 值 |
| --- | --- | --- |
| 燒結爐 | 設備代碼 | `QA-MCH-SIN-A001` |
| 燒結爐 | 設備名稱 | `驗證真空燒結爐 A` |
| 批次 | 批號 | `QA-BATCH-SIN-A001` |
| 批次 | 作業人員 | `QA Tester` |
| 品項 1 | 作業標準 | `QA-SPEC-A001` |
| 品項 1 | 數量 | `360` |
| 品項 2 | 作業標準 | `QA-SPEC-B001` |
| 品項 2 | 數量 | `80` |

### 5.5 操作步驟與預期結果

| 步驟 | 操作 | 預期結果 |
| --- | --- | --- |
| D-01 | 到「設備設定」新增真空式燒結爐 | 設備出現在真空式燒結爐列表 |
| D-02 | 到「真空式燒結作業」，未選爐直接送出 | 系統阻擋，提示需選擇真空式燒結爐 |
| D-03 | 選擇 `QA-MCH-SIN-A001`，輸入單一作業標準品項 | 推薦結果可產生，細節顯示高度與容量依據 |
| D-04 | 儲存燒結批次 `QA-BATCH-SIN-A001` | 真空式燒結批次列表新增資料 |
| D-05 | 加入第二個不允許混裝的作業標準 | 系統應提示混裝限制或避免合併不相容品項 |
| D-06 | 將墊塊堆疊數調到超過最大堆疊 | 系統應阻擋或在推薦細節標示不可行 |
| D-07 | 將產品高度調整為接近或超過爐層高度 | 推薦結果應顯示高度不符或不可行原因 |
| D-08 | 修改治具對應規則為限制後重新推薦 | 被限制治具不應被推薦為最佳方案 |
| D-09 | 到儀表板查看近期真空式燒結排程 | 可看到剛建立的批次 |
| D-10 | 到查詢報表以批號與日期查詢 | 可查到燒結批次與相關資訊 |

### 5.6 需要特別觀察

- 燒結與真空脫脂是否共用同一套 structured batch 行為，切換頁面時資料不應串台。
- 推薦結果是否能區分「可行」、「警告」、「不可行」。
- 治具關聯規則修改後，重新推薦是否立即反映。
- 儲存後的批次是否與推薦方案一致。

### 5.7 失敗判斷

- 不允許混裝的品項被直接合併且沒有提示。
- 超過高度限制仍顯示可行。
- 修改治具規則後推薦沒有變化。
- 儲存的批次與畫面推薦內容不一致。

## 6. 路線 E：資料生命週期、異常情境與穩定性驗證

### 6.1 故事情境

系統已經累積一批資料後，管理者開始編輯、停用、封存、恢復、搜尋與查報表，同時模擬錯誤輸入、重複資料、重開系統、資料刷新等狀況。此路線驗證系統面對長期使用與例外狀況的穩定度。

### 6.2 驗證目標

- 編輯、停用、封存、恢復不會破壞歷史資料。
- 已被引用的主檔不能被不安全刪除。
- 搜尋與報表可處理大量資料與不同條件。
- 重開系統、重新整理資料後狀態一致。
- 錯誤輸入會被阻擋，且提示清楚。

### 6.3 前置條件

- 路線 A 至 D 至少完成一次。
- 系統內已有產品、治具、設備、投入紀錄、真空批次與燒結批次。

### 6.4 操作步驟與預期結果

| 步驟 | 操作 | 預期結果 |
| --- | --- | --- |
| E-01 | 建立一筆重複產品代碼 | 系統阻擋或提示唯一性錯誤 |
| E-02 | 建立一筆重複治具編號 | 系統阻擋或提示唯一性錯誤 |
| E-03 | 將已被作業標準引用的產品主檔刪除或作廢 | 系統應封存或阻擋硬刪除，不破壞歷史資料 |
| E-04 | 恢復剛封存的產品主檔 | 主檔與下層規格可重新啟用或符合系統設計 |
| E-05 | 停用一台已存在批次的設備 | 歷史批次仍可查，新的作業不能誤用停用設備 |
| E-06 | 編輯已建立的作業標準高度或單盤容量 | 新推薦應採用新值，歷史批次不應錯亂 |
| E-07 | 刪除或停用一筆治具後重新推薦 | 系統不應推薦停用治具，既有歷史顯示仍可辨識 |
| E-08 | 在各搜尋視窗輸入不存在關鍵字 | 顯示 0 筆結果，不應報錯 |
| E-09 | 日期篩選設定未來日期 | 報表顯示 0 筆或只顯示符合資料 |
| E-10 | 快速切換多個頁籤 20 次 | 不應白屏、卡死、重複綁定事件造成多次送出 |
| E-11 | 儲存資料後立即切到其他頁再回來 | 資料應一致，不應因刷新時序遺失 |
| E-12 | 重開應用程式 | 已儲存資料完整保留 |
| E-13 | 在數字欄位輸入負數、小數或空白 | 依欄位限制阻擋或正確轉換，不應產生 NaN |
| E-14 | 測試 50 筆產品、50 筆治具、20 筆批次 | 搜尋、列表、頁籤切換仍可接受 |
| E-15 | 檢查系統管理與伺服器資訊顯示 | 不應出現空白、undefined 或錯誤資訊 |

### 6.5 需要特別觀察

- 事件監聽是否重複綁定：同一個按鈕按一次不應送出兩次。
- refresh 後目前頁籤與表單狀態是否合理。
- 封存與恢復是否連動到下層作業標準。
- 報表是否會因停用或封存資料而查不到歷史紀錄。

### 6.6 失敗判斷

- 編輯或封存資料造成歷史批次消失。
- 重複代碼可成功建立。
- 快速切換頁籤後按一次儲存卻產生多筆資料。
- 重開應用程式後資料遺失。
- 報表查詢出現 console error 或畫面空白。
