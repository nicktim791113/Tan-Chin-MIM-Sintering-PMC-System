## v1.4.0

本次更新文案：
> 分離桌面版與網頁版的預設連接埠。桌面版 `npm start` 內建 server 預設使用 `3105`，網頁版 `npm run server` 預設使用 `3106`。server-only 啟動時會在終端機顯示 `localhost` 與目前電腦可用的 LAN IP 網頁入口，方便內網端直接連線。

主要更新：
- `npm run server` 預設 port 改為 `3106`。
- `npm start` 桌面模式內建 server 預設 port 改為 `3105`。
- 新增 `PMC_WEB_PORT` / `PMC_WEB_HOST` 給網頁版 server-only 模式使用。
- 新增 `PMC_DESKTOP_SERVER_PORT` / `PMC_DESKTOP_SERVER_HOST` 給桌面版內建 server 使用。
- `/api/bootstrap` 與 `/api/system/meta` 回傳 `webEntryUrls`，包含目前可用的 `/web/` 入口清單。
- 終端機啟動 server-only 模式時會印出正確的 `http://目前電腦IP:3106/web/`。
- 版本由 `1.3.0` 調整為 `1.4.0`。

## v1.3.0

本次更新文案：

> 修正 `npm run server` 讀取錯誤舊資料庫路徑的問題。server-only 模式現在會預設使用與桌面版一致的 `%APPDATA%\tan-chin-mim-pmc-system\pmc-system.db`，避免誤讀 `%APPDATA%\Electron\pmc-system.db` 舊 schema。同時補上 server port 可調整方式，`3186` 只是預設值，可用 `PMC_SERVER_PORT` 或 `--port` 改成其他 port。

主要更新：

- 修正 `server.js` server-only 預設資料庫路徑。
- 新增 `server.js --port`、`server.js --host` 參數讀取。
- `npm start` 桌面模式也支援 `PMC_SERVER_PORT` / `PMC_SERVER_HOST`。
- 更新備份腳本與備份文件的 runtime 資料庫預設路徑。
- 版本由 `1.2.0` 調整為 `1.3.0`。

## v1.1.0

本次更新文案：

> 系統已建立桌面版與內網瀏覽器版共用畫面的基礎架構。`/web/` 會載入與 Electron 桌面版相同的 renderer，網頁版透過 HTTP API 操作同一套資料流程，讓後續版面、欄位、表單流程與文案可以同步維護。

主要更新：

- 新增 `/web/` 共用 renderer 入口。
- 新增瀏覽器版 API adapter。
- 新增 `/api/bootstrap`、`/api/system/meta`、`/api/system/logs`、`/api/reports/export`。
- 新增 `npm run server`，可只啟動 server 不開 Electron 視窗。
