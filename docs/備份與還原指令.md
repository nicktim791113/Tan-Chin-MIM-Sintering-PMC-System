# 備份與還原指令

更新日期：2026-05-19

## 建議備份時機

- 每次完成正式版本更新並 push GitHub 後。
- 內網多人操作上線前。
- 修改資料庫 schema、ERP 串接、報表匯出、權限或備份邏輯前。
- 現場正式資料庫已有重要資料時。

備份前建議先關閉 Electron 桌面版與 `npm run server`，避免 SQLite 還有寫入中的 WAL 檔。

## 預設備份指令

```powershell
npm run backup
```

預設輸出位置：

```text
backups/
```

備份檔名格式：

```text
Tan-Chin-MIM-Sintering-PMC-System-yyyyMMdd-HHmmss.zip
```

## 指定備份位置

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -DestinationRoot "D:\PMC-Backups"
```

## 指定 runtime 資料庫位置

預設會嘗試備份：

```text
%APPDATA%\Electron\pmc-system.db
```

若正式資料庫在其他位置，請使用：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -UserDataPath "C:\path\to\pmc-system.db"
```

## 只備份程式與文件，不備份 runtime 資料庫

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1" -SkipRuntimeData
```

## 備份內容

備份壓縮檔會包含：

- 核心程式檔：`main.js`、`server.js`、`database.js`、`preload.js`、`index.html`。
- 前端共用 renderer：`assets/`、`renderer-scripts/`。
- 文件：`docs/`、`README.md`。
- 專案設定：`package.json`、`package-lock.json`、`.gitignore`、`tailwind.config.js`。
- 備份腳本：`scripts/`。
- 可選 runtime 資料庫：`runtime-user-data/pmc-system.db`、`pmc-system.db-shm`、`pmc-system.db-wal`。
- `backup-manifest.json`。
- `RESTORE-INSTRUCTIONS.md`。

備份壓縮檔不包含：

- `node_modules/`
- `dist/`
- `.validation-data/`
- `.env` 或 `.env.*`

## 還原步驟

1. 先備份目前機器上的資料庫。
2. 關閉 Electron 桌面版與 `npm run server`。
3. 解壓縮備份檔到新資料夾。
4. 在新資料夾執行：

   ```powershell
   npm install
   ```

5. 若要還原 runtime 資料庫，將 `runtime-user-data/pmc-system.db` 還原到：

   ```text
   %APPDATA%\Electron\pmc-system.db
   ```

   或使用：

   ```powershell
   $env:PMC_DB_PATH="C:\path\to\pmc-system.db"
   npm run server
   ```

6. 桌面版啟動：

   ```powershell
   npm start
   ```

7. 只啟動內網 server：

   ```powershell
   npm run server
   ```
